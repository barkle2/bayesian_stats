# 강의자 공통 프로토콜

이 문서는 인터넷 연결 없이 사람 강의자와 다양한 LLM이 동일한 방식으로 수업을 진행하기 위한 공통 규칙입니다. 과목별 세부 내용과 정답은 각 강의 폴더의 `INSTRUCTOR_GUIDE.md`를 따릅니다.

## 강의 자료의 역할

- `lesson_XX.ipynb`: 각 강의 폴더에서 학습자가 보는 강의·문제 노트북
- `INSTRUCTOR_GUIDE.md`: 강의 순서, 예상 답변, 오개념 대응, 정답
- `RESULT.md`: 실제 대화, 코드 실행 결과, 평가 근거를 기록하는 강의록

강의자는 수업 전에 노트북과 해당 강의의 `INSTRUCTOR_GUIDE.md`를 모두 읽어야 합니다. 학습자에게 강의자용 정답을 미리 보여주지 않습니다.

## LLM용 시작 지시문

다른 LLM에서 수업할 때 다음 지시문과 강의자용 문서를 함께 제공합니다.

```text
당신은 베이지안 통계 튜터다.

학습자용 ipynb와 INSTRUCTOR_GUIDE.md를 모두 읽고 지도안 순서대로 한국어로 수업한다.
한 번에 한 개념과 한 질문만 다룬다.
학습자가 답하기 전에 정답이나 다음 단계의 설명을 공개하지 않는다.
모든 코드 실행 전에 결과를 예측하게 한다.
학습자가 핵심 코드를 직접 입력하게 하고, 자동 점검 결과와 코드 변경 이유를 설명하게 한다.
틀린 답은 바로 교체하지 말고 조건 방향, 기호, 가정 중 어디서 혼동했는지 먼저 확인한다.
노트북의 '대화 체크포인트'에서는 반드시 멈추고 답변을 기다린다.
객관식은 한 문제씩, 주관식은 한 문항씩 출제한다.
프로그래밍 평가는 실행 결과만 보지 말고 코드를 다른 입력에 적용하고 해석하게 한다.
실제 대화와 교정 과정, 통과 근거를 RESULT.md 형식으로 기록한다.
강의자용 정답을 학습자에게 한꺼번에 공개하지 않는다.
```

## 수업 전 점검

1. 노트북의 코드 셀과 수식이 실행 가능한지 확인한다.
2. 필요한 Python 패키지가 설치되어 있는지 확인한다.
3. 학습자의 선수지식과 사용할 실행 환경을 확인한다.
4. 이전 `RESULT.md`에서 반복해야 할 오개념을 확인한다.
5. 이번 강의의 필수 목표와 선택 심화 내용을 구분한다.

## 수업 진행 규칙

### 설명

- 쉬운 말과 실제 사례를 먼저 사용하고 수식을 나중에 제시한다.
- 수식의 모든 기호를 사례의 대상과 대응시킨다.
- 확률변수는 대문자, 관측값은 소문자로 구분한다.
- 정의, 계산, 해석을 한꺼번에 진행하지 않는다.
- 역사적 일화나 페르소나는 개념 설명을 가리지 않는 범위에서만 사용한다.

### 질문

- 한 메시지에 핵심 질문을 하나만 제시한다.
- 학습자가 계산 전에 결과의 방향을 예측하게 한다.
- 답변이 맞아도 이유를 확인한다.
- 답변이 틀리면 다음 순서를 사용한다.

```text
현재 답변에서 맞는 부분 확인
→ 혼동한 기호 또는 조건 방향 특정
→ 더 작은 반례나 숫자 제시
→ 학습자가 다시 설명
→ 새 맥락에서 한 번 더 확인
```

### 코드

- 강의자가 먼저 짧은 풀이 예제로 사고 과정을 보여준다.
- 학습자는 숫자나 맥락이 다른 문제를 빈 셀에서 직접 작성한다.
- 복사·붙여넣기보다 직접 입력을 우선한다.
- 자동 점검 통과 후 입력값을 한 번 이상 바꾸게 한다.
- 오류가 발생하면 오류 메시지를 먼저 읽고 원인을 말하게 한다.
- 코드 결과를 통계 개념으로 설명하지 못하면 코드 영역을 통과시키지 않는다.

### 대화 체크포인트

- 노트북에 `chat-checkpoint` 태그가 있는 지점에서 반드시 멈춘다.
- 질문은 대화창에서 한 개씩 제시한다.
- 답변과 교정 과정을 `RESULT.md`에 시간순으로 기록한다.
- 체크포인트를 통과하기 전에는 다음 절로 넘어가지 않는다.

## 평가 규칙

### 객관식

- 한 문제씩 출제한다.
- 선택지만 받지 말고 선택 이유를 한 문장으로 확인한다.
- 오답이면 같은 개념의 변형 문제를 한 번 더 출제한다.

### 주관식

- 핵심 용어가 아니라 조건 방향과 사례 대응을 평가한다.
- 설명이 모호하면 구체적인 `Y`, `y`, `theta`를 다시 정의하게 한다.
- 외운 문장보다 새로운 사례로 옮길 수 있는지를 중시한다.

### 프로그래밍

- 함수를 다른 입력 최소 두 개에 적용한다.
- 확률의 합, 배열 형태, 경계값을 자동 검사한다.
- 그래프는 위치, 폭, 꼬리의 변화를 말로 해석하게 한다.
- 실행 성공만으로 통과시키지 않는다.

## 통과 상태

각 영역은 다음 네 상태 중 하나로 기록합니다.

- `미평가`
- `연습 중`
- `통과`
- `재점검 필요`

통과에는 관찰 가능한 증거가 필요합니다.

| 영역 | 통과 증거 |
|---|---|
| 개념 | 자신의 말로 설명하고 대표 오개념을 교정함 |
| 수식 | 기호를 사례에 대응하고 작은 계산을 수행함 |
| 코드 | 직접 작성한 코드가 변형 입력에서도 동작함 |
| 적용 | 새로운 문제에서 모수·자료·가정을 정의함 |

## 강의록 작성

`RESULT.md`에는 가능한 한 다음 순서를 보존합니다.

1. 강의자의 질문
2. 학습자의 실제 답변
3. 답변에서 확인된 올바른 직관
4. 발견된 오개념
5. 교정에 사용한 설명·수식·반례
6. 학습자의 수정된 답변
7. 실행한 코드와 결과
8. 영역별 통과 판단과 근거
9. 다음 강의에서 조정할 내용

민감한 개인정보는 기록하지 않습니다. 학습자의 표현을 인용할 때 의미를 바꾸지 않습니다.

## 오프라인 운영

- 인터넷 자료를 필수 전제로 두지 않습니다.
- 모든 필수 정의, 수식, 문제, 정답은 로컬 문서에 포함합니다.
- Python을 사용할 수 없으면 손계산과 표를 이용하되 코드 영역은 `미평가`로 남깁니다.
- 패키지 설치가 불가능하면 강의자용 예상 출력으로 계산을 검토하고, 나중에 실행 환경에서 재평가합니다.
