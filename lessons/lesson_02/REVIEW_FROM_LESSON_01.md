# 1강에서 가져온 복습 포인트

2강을 시작하기 전에 1강에서 확인된 강점과 반복할 지점을 정리합니다.

## 안정적으로 이해한 내용

- 사전분포, 우도, 사후분포의 기본 역할
- 유한 가설에서 `사전 × 우도 → 정규화` 계산
- 사후분포를 다음 갱신의 사전분포로 사용하는 과정
- \(p\), \(Y\), \(y\)의 기본 구분
- Beta-Binomial 켤레 갱신
- 사전 유효 표본 크기와 데이터 수가 사후평균에 미치는 영향
- 데이터가 많아질수록 사후분포의 폭이 줄어든다는 직관
- 노동시장 재취업률 문제에 Beta-Binomial 갱신을 적용하는 방법

## 2강에서 반복할 내용

### 1. 우도는 모수에 대한 확률분포가 아니다

\[
P(Y=y\mid\theta)
\]

는 \(\theta\)가 주어졌을 때 데이터 \(y\)가 관측될 확률입니다. 관측값 \(y\)를 고정하고 \(\theta\)를 움직이면 우도 함수가 되지만, 그 자체가 \(\theta\)에 대한 확률분포는 아닙니다.

### 2. 조건 방향을 뒤집지 않는다

\[
P(Y=y\mid\theta)
\]

와

\[
P(\theta\mid Y=y)
\]

는 서로 다른 질문에 답합니다.

- \(P(Y=y\mid\theta)\): 이 모수가 맞다면 이런 데이터가 얼마나 잘 나오는가?
- \(P(\theta\mid Y=y)\): 이런 데이터를 본 뒤 이 모수를 얼마나 믿는가?

### 3. 연속형 모수에서는 한 점의 확률과 밀도를 구분한다

연속형 \(\theta\)에서 \(P(\theta=0.5\mid y)\)는 보통 0입니다. 대신 다음을 봅니다.

- \(0.49 < \theta < 0.51\) 같은 구간확률
- \(\theta=0.5\) 부근의 사후밀도
- 사후평균, 중앙값, MAP, credible interval

### 4. `successes`, `failures`, `n`, `y`를 구분한다

1강 마지막 문제에서 한 번 다음처럼 입력한 적이 있습니다.

```python
beta_update(3, 7, 20, 12)
```

하지만 함수가 `successes, failures`를 받는다면 이는 “성공 20명, 실패 12명”이라는 뜻입니다.

2강에서는 일부 함수가 의도적으로 `n, y`를 받도록 설계됩니다.

```python
def beta_binomial_update(alpha_prior, beta_prior, n, y):
    successes = y
    failures = n - y
```

이렇게 하면 전체 수 \(n\), 성공 수 \(y\), 실패 수 \(n-y\)의 역할을 더 명확히 확인할 수 있습니다.

### 5. 같은 성공률이어도 표본 수가 다르면 폭이 다르다

다음 두 자료는 관측 성공률이 모두 0.6입니다.

```text
A: n = 20,  y = 12
B: n = 200, y = 120
```

하지만 B가 훨씬 더 많은 정보를 줍니다. 따라서 B의 우도와 사후분포는 A보다 더 좁아집니다.

### 6. 모델에 없는 값은 데이터가 많아도 자동으로 생기지 않는다

유한 가설을 \([0.2, 0.5, 0.8]\)로 정했다면, 실제 값이 0.6에 가깝더라도 사후분포는 세 후보 중에서만 확률을 나눕니다.

이 문제를 해결하려면 더 촘촘한 격자나 연속형 사전분포를 사용해야 합니다.

## 2강 시작 전 빠른 질문

3. `n=20`, `y=12`일 때 성공 수와 실패 수는 각각 몇 명인가요?
