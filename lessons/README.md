# 대화형 강의 운영 원칙

이 폴더의 강의는 설명을 읽고 완성 코드를 실행하는 방식이 아니라, 학습자가 예측하고 직접 구현하면서 개념을 확인하는 방식으로 진행합니다.

## 강의자용 문서

- [공통 강의자 진행 프로토콜](INSTRUCTOR_PROTOCOL.md): 모든 강의에 적용하는 질문 방식, 코드 지도 원칙, 평가 및 기록 방법
- 각 강의 폴더의 `INSTRUCTOR_GUIDE.md`: 해당 노트북의 진행 순서, 예상 답변, 오개념 대응, 정답과 채점 기준

LLM은 공통 프로토콜을 먼저 읽고 해당 강의의 상세 강의록을 이어서 읽습니다. 인터넷을 사용할 수 없는 환경의 사람 강의자도 같은 순서로 사용하면 됩니다.

## 한 강의의 기본 구조

각 강의 노트북은 다음 순서로 구성합니다.

1. **개념 강의**: 새 기호와 개념을 실제 사례에 대응해 설명한다.
2. **풀이 예제**: 손계산과 완성된 짧은 코드의 사고 과정을 보여준다.
3. **대화 체크포인트**: 튜터가 대화창에서 한 문제를 묻고 답을 기다린다.
4. **직접 풀이**: 숫자나 맥락을 바꾼 문제를 학습자가 직접 계산한다.
5. **직접 코딩**: 핵심 코드를 빈 셀에 입력한다.
6. **자동 점검**: 값, 형태, 확률의 합, 입력 변경 내성을 검사한다.
7. **코드 변형**: 데이터·사전분포·모형 가정을 바꾸고 결과를 설명한다.
8. **응용**: 노동시장 또는 새로운 문제로 같은 구조를 옮긴다.
9. **종합 평가**: 객관식, 주관식, 프로그래밍 문제로 숙달을 확인한다.

강의 설명만 이어가거나 처음부터 빈 코드 셀만 제시하지 않습니다. 먼저 풀이 사고를 시범으로 보인 뒤, 구조가 같은 새 문제를 학습자가 해결하게 합니다.

## 기본 학습 루프

모든 핵심 개념은 다음 순서를 따릅니다.

1. **예측**: 코드를 실행하기 전에 결과의 방향과 이유를 말한다.
2. **직접 입력**: 핵심 계산이나 모델 코드를 빈 셀에 직접 작성한다.
3. **실행**: 결과와 오류 메시지를 관찰한다.
4. **자동 점검**: 별도의 검사 셀로 값, 형태, 확률의 합 등을 확인한다.
5. **설명**: 결과가 나온 이유를 자신의 말로 설명한다.
6. **변형**: 데이터, 사전분포, 모형 가정을 한 가지씩 바꾼다.
7. **비교**: 변경 전후 결과를 수치와 그래프로 비교한다.
8. **전이**: 같은 구조를 노동시장 또는 새로운 문제에 적용한다.

## 노트북 구성

- 풀이 예제에는 짧은 완성 코드와 해설을 제공합니다.
- `직접 입력` 셀에는 풀이 예제와 숫자나 맥락이 다른 문제를 두고 완성 답안을 미리 넣지 않습니다.
- 바로 아래의 `자동 점검` 셀은 학습자가 작성한 변수와 함수를 검사합니다.
- 막혔을 때만 열어볼 수 있도록 힌트를 단계별로 제공합니다.
- 정답 코드는 학습을 마친 뒤 비교할 수 있도록 별도 해설에 둡니다.
- 단순 복사 방지를 위해 같은 개념을 다른 숫자와 데이터로 다시 구현합니다.

## 대화 체크포인트

- 노트북의 `대화 체크포인트`에 도달하면 튜터가 대화창에서 해당 문제를 한 개씩 묻습니다.
- 학습자가 답하기 전에는 다음 개념으로 넘어가지 않습니다.
- 틀린 답은 즉시 정답으로 덮지 않고, 어떤 조건 방향이나 기호에서 혼동했는지 확인합니다.
- 대화 결과는 해당 강의의 `RESULT.md`에 시간순으로 기록합니다.
- 노트북은 학습 자료와 코드 실습을 담당하고, 대화창은 문답과 피드백을 담당합니다.

## 종합 평가

강의 마지막에는 다음 평가를 모두 수행합니다.

- **객관식**: 핵심 개념과 흔한 오개념을 구분한다.
- **주관식**: 수식의 기호를 사례에 대응하고 결과를 자신의 말로 해석한다.
- **프로그래밍**: 빈 셀에서 함수를 작성하고 변형 데이터에 적용한다.

객관식과 주관식은 대화창에서 한 문제씩 답하고, 프로그래밍 문제는 노트북에서 작성한 뒤 실행 결과와 해석을 대화창에 제출합니다.

## 코드 숙달 기준

다음 조건을 모두 만족해야 코드 영역을 통과합니다.

- 핵심 코드를 직접 입력했다.
- 자동 점검을 통과했다.
- 입력값을 바꾸어도 코드가 동작한다.
- 결과가 바뀐 방향을 개념으로 설명했다.
- 새 사례에 같은 코드를 적용했다.

## 대화형 강의록

각 강의의 `RESULT.md`에는 다음을 기록합니다.

- 튜터의 질문과 학습자의 실제 답변
- 답변에서 드러난 직관과 오개념
- 오개념을 교정한 설명과 수식
- 학습자가 수정한 최종 설명
- 실행한 코드와 확인한 결과
- 영역별 통과 근거
- 다음 강의에서 조정할 난이도와 교수법

가능한 한 대화 순서를 보존하여, 나중에 무엇을 어떻게 이해하게 되었는지 다시 추적할 수 있게 합니다.
